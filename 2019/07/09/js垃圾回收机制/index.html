垃圾回收机制

本文最后更新于：2021年3月24日 晚上

javascript具有自动垃圾收集机制

执行环境会负责管理代码执行过程中使用的内存\

垃圾收集器是周期性运行的，触发垃圾收集的阈值是动态设定的

方法

标记清除

  • 最主流的垃圾收集算法
  • 离开作用域，没有被引用的值被标记为可以回收。在垃圾收集期间被删除

引用计数

  • 跟踪记录被引用的次数
  • 次数为0的值所占的内存会被下次垃圾收集回收

循环引用

objA.obj = objB
objB.obj = objA
a/b中都包含指向对方的指针

手动解除变量的引用

  • 当数据不再有用，将其设置为null，释放其引用
  • 解除了引用的值脱离了执行环境，下次垃圾收集期间会被回收

内存泄漏

内存泄露是指一块被分配的内存既不能使用，又不能回收，直到浏览器进程结束。

引起内存泄漏的情况

  1. 全局变量

  2. 闭包

  3. 循环引用

  4. 未清除的定时器

  5. 控制台日志

ES6中的weak

weakset和weakmap不被垃圾收集机制的引用计数
