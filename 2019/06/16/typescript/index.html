TypeScript

本文最后更新于：2021年4月8日 晚上

TypeScript is a typed superset of JavaScript that compiles to plain JavaScript. ts是js的超集，相比js添加了可选的静态类型和基于类的面向对象编程

数据类型

  • string(字符串)
  • number(数字)
  • boolean(布尔)
  • array(数组)
  • tuple(元祖)
  • enum(枚举)
  • any(任意)
  • null
  • undefined
  • void
  • never

类型校验

不按照定义的数据类型赋值报错

array/数组

//第一种
var arr1:number:[] = [1,2,3]
//第二种
var arr2:Array<number> = [1,2,3]

  • 数组的项中不能出现其他类型

    let fibonacci: number[] = [1, 1, '2', 3, 5];
// = 后的类型被推断为number|string，前面是number，报错

  • 数组的一些方法的参数类型也会受到限制

    number[]的数组中push('abc')报错
// abc是字符串

  • 数组泛型

  • 用接口表示数组

  • any/任意类型的数组

    let list: any[] = ['Xcat Liu', 25, { website: 'http://xcatliu.com' }];

  • 类数组/Array-like Object
    常见的类数组都有自己的接口定义

tuple/元组

元组按一定顺序预定义数组结构中的重数据类型

let tuple1: [number,string,boolean];
tuple1 = [666,'字符串',true]

enum/枚举

实现考虑到某一变量可能的值，尽量用自然语言中含义清楚的单词来表达他的每一个值。这种方法叫做枚举方法，用这种方法定义的数据类型

enum Flag  {
  success = 1,
  error = 2
}
let s:Flag = Flag.success
console.log(s) // 1
// 未赋值则打印下标

任意值类型 any

ts中的普通类型在赋值过程中不能改变类型

let myFavoriteNumber: string = 'seven';
myFavoriteNumber = 7;

// index.ts(2,1): error TS2322: Type 'number' is not  assignable to type 'string'.

// any类型就可以改变
let myFavoriteNumber: any = 'seven';
myFavoriteNumber = 7;
  • 任意值上可以访问任何属性和方法，
  • 声明一个变量为任意值之后，对它的任何操作，返回的内容的类型都是任意值
  • 未声明类型的变量，自动给识别为任意类型
    let something;
something = 'seven';
something = 7;

something.setName('Tom');

其他类型/never

表示从不会出现的值

var a:never
a = (()=>{
  throw new Error('错误')
})

null/undefined是never的子类型

var num1:undefined
var num2:undefined | number
num = 2
console.log(num2) //2

var num3:null
num = null // ok
num = 其他都报错

void

一般用于定义的方法没有return

function yell():void{
  console.log('666')

类型推论

  • 如果没有明确的指定类型，那么 TypeScript 会依照类型推论（Type Inference）的规则推断出一个类型。 

    let myFavoriteNumber = 'seven';
// 被自动推测为string类型
myFavoriteNumber = 7;

// index.ts(2,1): error TS2322: Type 'number' is not assignable to type 'string'.

  • ！！！！未声明类型的变量！！！！！
    自动给识别为任意类型
    不会被类型检查推测

联合类型

let myFavoriteNumber: string | number;
myFavoriteNumber = 'seven';
myFavoriteNumber = 7;
  • 可以访问联合类型的共有的属性或方法
  • 联合类型的变量在赋值的时候，一样会被推断出一个类型
function getString(something: string | number): string {
  return something.toString();
}
let myFavoriteNumber: string | number;
myFavoriteNumber = 'seven';
console.log(myFavoriteNumber.length); // 5
myFavoriteNumber = 7;
console.log(myFavoriteNumber.length); // 编译时报错

  • 类型别名常用于联合类型

    type Name = string;
type NameResolver = () => string;
type NameOrResolver = Name | NameResolver;
function getName(n: NameOrResolver): Name {
  if (typeof n === 'string') {
    return n;
  } else {
    return n();
  }
}

对象的类型-接口/interfaces

  • 在面向对象语言中，接口（Interfaces）是一个很重要的概念，它是对行为的抽象，而具体如何行动需要由类（classes）去实现（implement）。

  • TypeScript 中的接口是一个非常灵活的概念，除了可用于对类的一部分行为进行抽象以外，也常用于对「对象的形状（Shape）」进行描述。

    interface Person {
name: string;
age: number;
}
let xiaoming: Person = {
name: 'xiaoming',
age: 66
}

  • 赋值时，变量的形状必须和接口的形状保持一致，属性不多也不少

    • 可选属性
      age?: number;
//age可有可无
    • 任意属性
      一旦定义了任意属性，那么确定属性和可选属性的类型都必须是它的类型的子集
      interface Person {
name: string;
age?: number;
[propName: string]: string;
}

let tom: Person = {
    name: 'Tom',
    age: 25,
    gender: 'male'
};
//age?-->25是number-->number不是[propName: string]: string;的子集

// index.ts(3,5): error TS2411: Property 'age' of type    'number' is not assignable to string index type 'string'.
// index.ts(7,5): error TS2322: Type '{ [x: string]: string |     number; name: string; age: number; gender: string; }' is not    assignable to type 'Person'.
//   Index signatures are incompatible.
//     Type 'string | number' is not assignable to type     'string'.
//       Type 'number' is not assignable to type 'string'.
      • 只读属性
        只能在创建的时候被赋值
        interface Person {
readonly id: number;
name: string;
age?: number;
[propName: string]: any;
}

let tom: Person = {
  id: 89757,
  name: 'Tom',
  gender: 'male'
};

tom.id = 9527;

// index.ts(14,5): error TS2540: Cannot assign to 'id' because it is a constant or a read-only property.

  • type 和 interface的区别

    type Name = string
// 类型别名
interface Human1{
  name:string
}
interface Human2{
  age:number
}
type Person = Human1 | Human2
// 联合类型
type PersonArr = [Human1,Human2]
// 定义具体位置的类型


interface User {
  name: string
  age: number
}

interface User {
  sex: string
}

/*
User 接口为 {
  name: string
  age: number
  sex: string 
}
*/
// 两个同名interface声明会合并

    函数的类型

    • 函数声明

      function sum(x: number, y: number): number {
  return x + y;
}
// 输入的参数不能多也不能少

    • 函数表达式
      ！！！这里不是箭头函数

      let mySum: (x: number, y: number) => number{
  return x + y
}
let mySum = function (x: number, y: number): number {
  return x + y;
};
//在 TypeScript 的类型定义中，=> 用来表示函数的定义，左边是输入类型，需要用括号括起来，右边是输出类型。

    • 用接口定义函数的形状

      interface SearchFunc {
  (source: string, subString: string): boolean;
}

let mySearch: SearchFunc;
mySearch = function(source: string, subString: string) {
  return source.search(subString) !== -1;
}

    • 可选参数和对象的可选属性一样，加?

      // 可选参数只能配置到参数们的末尾
firstName?: string, lastName: string
// 报错！！！！


firstName: string, lastName?: string
// ok

    • 参数默认值

      (firstName: string, lastName: string = 'Cat')
      识别为可选参数
      不再受可选择参数必须放在必须参数后面的限制

    • 函数重载

      • 不同输入输出选项的函数，可以重复定义多次，把精确的定义写在前面
      • TypeScript 会优先从最前面的函数定义开始匹配
      function reverse(x: number): number;
function reverse(x: string): string;

function reverse(x: number | string): number | string {
  if (typeof x === 'number') {
    return Number(x.toString().split('').reverse().join(''));
  } else if (typeof x === 'string') {
    return x.split('').reverse().join('');
  }
}

类型断言

  • 在不确定类型的时候就访问方法或者属性会报错
  • 此时使用类型断言，在需要断言的变量之前加上
  • 断言只能存在联合类型中的类型
function getLength(something: string | number): number {
  if ((<string>something).length) {
      return (<string>something).length
  } else {
      return something.toString().length
  }
}

声明语句

  • declare var 声明全局变量
  • declare function 声明全局方法
  • declare class 声明全局类
  • declare enum 声明全局枚举类型
  • declare namespace 声明（含有子属性的）全局对象
  • interface 和 type 声明全局类型
  • export 导出变量
  • export namespace 导出（含有子属性的）对象
  • export default ES6 默认导出
  • export = commonjs 导出模块
  • export as namespace UMD 库声明全局变量
  • declare global 扩展全局变量
  • declare module 扩展模块
  • /// 三斜线指令
  • 使用 @types 统一管理第三方库的声明文件。
