1224刷题

  1. 浏览器缓存机制

    • 强缓存

      1. 浏览器访问本地缓存直接验证看是否过期，如果没过期直接使用本地缓存，并返回code 200
      2. 服务端在响应头中添加Cache-Control/Expires/Pragma字段来为资源设置有效期
      3. Cache-Control设置有效秒数
      4. Expires设置有效时间戳
      5. Pragma:no-cache效果等于Cache-Control:no-cache跳过本地资源强缓存验证进行协商缓存验证
      6. 优先级：pragma -> cache-control -> expires

    • 协商缓存

      1. 强制缓存失效后，浏览器询问服务器是否继续使用当前缓存的过程

      2. 协商缓存生效，返回304和Not Modified

      3. 协商缓存失效，返回200和新资源

      4. 两种HTTP header方案

        1. Last-Modified（服务端）和If-Modified-Since（客户端请求携带），对比两个时间戳（一秒钟内发生的修改则会丢失精度！）

        2. Etag（服务器返回的唯一标识符，资源有变化会生成新的Etag）和If-None-Match（客户端请求携带），对比两个标示是否相同

    • 优先级

      1. Pragma->Cache-Control->Expires
      2. 全部不符合或者no-cache则进去协商缓存
      3. 服务器同时判断ETag和Last-Modified

    • 浏览器行为

      1. 地址栏访问，链接跳转是正常用户行为，将会触发浏览器缓存机制；
      2. F5刷新，浏览器会设置max-age=0，强缓存失效，进行协商缓存判断；
      3. ctrl+F5刷新，跳过强缓存和协商缓存，直接从服务器拉取资源。

  2. CSS3动画animation/translate/transition/transform

    • transform 属性向元素应用 2D 或 3D 转换
    • translate() 在xyz轴上移动元素
    • transition css过渡动画
      • transition-property 名称
      • transition-duration 时间
      • transition-timing-function 时间曲线
      • transition-delay 延迟
    • animation 关键帧动画
      • animation-name：动画名称
      • animation-duration: 间隔时间
      • animation-timing-function: 时间曲线
      • animation-delay: 延迟
      • animation-iteration-count：动画次数
      • animation-direction: 方向
      • animation-fill-mode: 规定对象动画时间之外的状态

  3. 盒模型

    • 在一个文档中，每一个元素都被表示为矩形的盒子，渲染引擎需要确定每个盒子的尺寸，属性和位置
    • css使用盒模型来描述每一个盒子所占的空间
    • 盒模型分为：
      1. 标准盒模型content-box：设置宽高–>content的宽高
      2. IE盒模型border-box：设置宽高–>整个盒子的大小（content+padding+border）

  4. css选择器的优先级

    !important > 行内样式 > id > class > 标签 > * > 继承 > 默认

  5. forEach/map/filter

    • forEach遍历数组调用回调函数，无返回，会改变原数组个数组；
    • map遍历数组调用回调函数，其每一项的返回值组成新的数组，不改变原数组
    • filter会返回原数组的一个子集，回调函数用于逻辑判断，返回 true则将当前元素添加到返回数组中，否则排除当前元素，原数组不变

function getType(obj){
  if(obj === null) return String(obj)
  return typeof obj === 'object' ?  Object.prototype.toString.call(obj).replace('[object ','').replace(']','').toLowerCase() : typeof obj
}
