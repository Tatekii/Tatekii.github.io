devServer.proxy对象

本文最后更新于：2019年8月6日 下午

该功能由http-proxy-middleware提供

开发环境中经常会用ajax获取假数据模拟渲染页面

methods: {
  getHomeInfo () {
    axios.get('/static/testData/index.json')
      .then(this.getHomeInfoSucc)
  },
  getHomeInfoSucc (res) {
    console.log(res)
  }
}

项目上线时，api接口都要写成api的格式

axios.get('/api/index.json')

实际生产中在项目上线之前修改代码是很高风险的操作

所以我们可以使用代理来解决，即在开发环境下给url路径设置代理

// 配置该代理在config/index.js
dev: {
...
    proxyTable: {
      //  请求 /api 都会被转发到target下的路径
      '/api': {
        target: 'http://localhost:3000',
        // 并且改写路径
        pathRewrite: {
          '^/api': '/static/data'
          // 最终请求地址就是http://localhost:8080/static/data
          // 也可以置空 改写 pathRewrite
        }
      }
    }
...


// 多个代理可以使用数组
proxy: [{
  context: ["/auth", "/api"],
  target: "http://localhost:3000",
}]


//默认情况下，不接受运行在 HTTPS 上，且使用了无效证书的后端服务器。如果你想要接受，修改配置如下：
proxy: {
  "/api": {
    target: "https://other-server.example.com",
    secure: false
  }
}


// 有时你不想代理所有的请求。可以基于一个函数的返回值绕过代理
proxy: {
  "/api": {
    target: "http://localhost:3000",
    // 在函数中你可以访问请求体、响应体和代理选项。必须返回 false 或路径，来跳过代理请求。

   //对于浏览器请求，你想要提供一个 HTML 页面，但是对于其他 API 请求则保持代理。你可以这样做：
    bypass: function(req, res, proxyOptions) {
      if (req.headers.accept.indexOf("html") !== -1) {
        console.log("Skipping proxy for browser request.");
        return "/index.html";
      }
    }
  }
}

// 受到跨域限制时
proxy: {
      '/api': 'http://localhost:3000',
      changeOrigin: true
      // 改变请求的源
    }
