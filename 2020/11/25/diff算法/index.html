diff算法

本文最后更新于：2021年3月24日 下午

概念

diff算法，包含渲染器如何对各种类型的vNode的属性，text，fragment进行patch更新，以及判断是patch vNode，还是新增还是删除

简单diff

  • newNode.children.length = 0 => 删除oldNode.children
  • oldNode.children = 0 => 新增newNode.children
  • 添加全部子节点/删除全部子节点

核心diff

  • 相对于简单diff就是优化对子节点的操作过程
  • 尽可能复用节点

key的作用

  • vNode中的唯一标识符
  • 用于保存新旧vNode的映射关系
  • 找到可复用的vNode

移动节点->最大索引值k

  • 新旧vNode，依次取出新vNode去旧vNode中寻找正确位置

  • 如果索引呈现递增趋势
    （newA->oldA=>0,newB->oldB=>1,newC->oldC=>2）则说明新旧 children 中节点顺序相同，不需要移动操作。相反的，如果在寻找的过程中遇到的索引值不呈现递增趋势，则说明需要移动操作

  • 如果匹配过程中递增规律被打破

    k为在旧 children 中所遇到的最大索引值，则后续匹配的索引小于这个值意味着都需要移动

vue2的双端比较

  • vue同时从新旧children的两端开始比较
  • 四个指针，分别查找是否key相同
  • 有相同则匹配项目的两个指针向内移动，对匹配成功的vNode进行patch
  • 新children中拥有新的vNode时，会发生匹配不上的情况，新增节点
  • 匹配结束后旧children中还有未处理的节点，删除

vue3的inferno

  • 文本diff->排除相同的前置和后置元素

    a,b,c
| | |
-------
|   | |
a,e,b,c
    • 分别从新旧节点的头和尾出发
    • 依次遍历到key值不一样时停止
    • 这样匹配成功后剩下的部分就是需要新增/删除的vNode

  • 文本diff后，使用剩余新children的长度建立source索引数组

    oldChildren = [b,c,d,f]
newChildren = [c,d,b,g]
source = [-1,-1,-1,-1]
// 默认全为-1
// 如果在旧children中匹配到key，则把该位置的-1改为旧vNode的索引，并且patch该节点
// 比如
source = [1,2,0,-1]

  • 使用最大索引算法判断节点是否需要移动

  • 根据source求出最长递增子序列

    LIS = [0，1]
// LIS中存储的是source索引
// 表示新旧children中这几个节点的先后关系相同

  • 根据LIS得出不进行移动的节点

    [c,d]

  • 从LIS和新children尾处建立指针处理余下节点

最长上升子序列

  • 子序列：不要求连续
  • 字串：要求连续
// 动态规划
const longestChildSequence = arr => {
  let len = arr.length
  if(len < 1) return
  const dp = Array(len).fill(1)
  // dp[i]代表原始数组该位置的最长子序列长度
  for(let i = 0 ; i < len ; i++){
    for(let j = 0 ; j < i ; j++){
      // j遍历之前遍历过的所有元素
      // 可能存在大小不一的递增序列，使用max取得最大值
      if(arr[i] > arr [j]){
        dp[i] = Math.max(dp[i],dp[j]+1)
        // [1,2,3,4,1,2,5]
        // 如果只使用dp[j]+1 ，dp[6]会在[1,2,5]中得出结果3
        // 使用max比较之前更长的序列[1，2，3，4，5]得到5

      }
    }
  }
  return Math.max(...dp)
}

// 贪心+二分
const longestChildSequence = arr => {
  let len = arr.length
  if(len < 1) return
  const res = [arr[0]]
  //将arr中后续元素arr[i]与res末尾元素比较
  //大于则push进res
  //小于则查找res中第一个比arr[i]大的元素并替换
  //由于存在替换，子序列值不一定正确，但长度是正确的
  for(let i = 0 ; i< len ;i++){
    if(arr[i]>res[res.length-1]){
      res.push(arr[i])
    }else{
      // res中左右指针的index
      // 二分查找
      let left = 0
      let right = res.length - 1
      while(left<right){
        let mid = Math.floor((left+right)/2)
        // let mid = ((left+right)/2) | 0
        // 浮点小数向下求整
        if(arr[i]>res[mid]){
          left = mid + 1
        }else{
          right = mid
        }
      }
      res[left] = arr[i]
    }
  }
  return res.length 
}
