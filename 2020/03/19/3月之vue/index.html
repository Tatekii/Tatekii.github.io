  • 生命周期

    1. beforeCreate

      【创建前】实例化了一个空的vue实例对象，只有最基本的生命周期函数和方法，在数据观测 (data observer) 和 event/watcher 事件配置之前被调用，data，prop数据和methods都是获取不到的（这些数据在initState()中初始化）

    2. created

      【创建后】数据和方法的初始化已经完成

      • 如果要调用methods的方法或者修改data数据最早只能在created中操作
      • 进行Ajax数据的请求获取

    3. beforeMount

      【挂载前】内存中的模版已经编译好了，还没有挂载在页面中

    4. Mounted

      【挂载后】将内存中的模版替换到浏览器页面中

      • 只触发一次
      • 与dom相关的操作，最早只能在mounted中进行
      • mounted执行完了就代表vue实例已经初始化完毕，进入运行阶段
      • 可以配合$.nextTick 使用进行单一事件对数据的更新后更新dom

    5. keep-alive组件中独有的

      • activated

        • activated在组件第一次渲染时会被调用，之后在每次缓存组件被激活时调用。

      • deactivated

        • 使用了keep-alive就不会调用beforeDestroy(组件销毁前钩子)和destroyed(组件销毁)，因为组件没被销毁，被缓存起来了。

      • 缓存了组件之后，再次进入组件不会触发beforeCreate、created 、beforeMount、 mounted

      • keep-alive的作用主要是在组件切换时，保存组件的状态，防止重复渲染引发性能问题。

    6. beforeUpdate

      【更新前】data数据是新的，页面数据还是旧的

      • vdom re-render and patch以新的data数据在内存中渲染新的模版,新旧模版进行patch对比，将要更新的dom渲染到页面中
      • 可以在更新前访问现有dom，如手动移除添加的事件监听器；

    7. updated

      【更新后】data数据和页面已经是同步的了

      • 此步骤不可操作数据，会进入死循环

    8. beforeDestroy

      【销毁前】执行beforeDestroy钩子，vue实例就从运行阶段进入销毁阶段

      • 执行beforeDestroy的时候，实例身上的所有功能数据都还在，没有进行真正的销毁过程
      • 之后就开始销毁，如果有子组件也会被销毁
      • 添加销毁确认的提示
      • 在实例销毁之前可用于销毁定时器，插件对象，解绑一些使用addEventListener 监听的事件等

    9. destroyed

      【销毁后】销毁结束才会执行destroyed，组件内的啥啥啥都不可用了

      • 此时 vue 实例已经解除了事件监听以及和 dom 的绑定，但是 dom 结构依然存在

  • 列表渲染件中key的作用

    • 优化diff性能(复用DOM)

      修改[a,b,c,d]->[a,b,666,c,d]
没有key会从b节点后全部删除再新建
有key的话key值能记录位置顺序，在正确的位置插入节点

    • 保证有状态组件的状态正确

      举个例子：一个新闻列表，可点击列表项来将其标记为”已访问”，可通过tab切换“娱乐新闻”或是“社会新闻”。

      不带key属性的情况下，在“娱乐新闻”下选中第二项然后切换到“社会新闻”，”社会新闻”里的第二项也会是被选中的状态，因为这里复用了组件，保留了之前的状态。【只更新了标签内容没有重新渲染标签】要解决这个问题，可以为列表项带上新闻id作为唯一key，那么每次渲染列表时都会完全替换所有组件，使其拥有正确状态。
